### 1.2 两条核心数据流

| 数据流 | 方向 | 说明 |
|--------|------|------|
| ① 知识同步 | 聚宝赞 → 我方 | 聚宝赞知识变更时，调用我方接口推送数据，我方负责切片、向量化、入库 |
| ② 消费者咨询 | 聚宝赞 ↔ 我方 | 聚宝赞转发消费者消息，我方识别意图后路由处理（检索知识库 / 回调聚宝赞业务 API），SSE 流式返回回复 |

### 1.3 我方暴露的接口清单

| 序号 | 接口 | 方法 | 用途 |
|------|------|------|------|
| 1 | `/api/v1/chat/{tenant_id}/stream` | POST | 消费者对话（SSE 流式） |
| 2 | `/api/v1/chat/{tenant_id}/history/{session_id}` | GET | 获取历史消息 |
| 3 | `/api/v1/knowledge/sync/{tenant_id}/{kb_type}` | POST | 全量/增量同步知识库 |
| 4 | `/api/v1/knowledge/sync/{tenant_id}/{kb_type}/batch` | POST | 批量增删知识 |
| 5 | `/api/v1/knowledge/sync/{tenant_id}/{kb_type}` | DELETE | 清空知识库 |

## 2. 通用规范

### 2.1 通信协议

- 协议：HTTPS（生产环境）/ HTTP（联调环境）
- 字符编码：UTF-8
- 数据格式：JSON（`Content-Type: application/json`）
- 时间格式：ISO 8601（`2026-06-02T15:30:00`）

### 2.2 认证方式

所有接口调用需携带 JWT Token（由聚宝赞 auth-service 签发，HS256 签名）。

| 环境 | 认证方式 | 说明 |
|------|---------|------|
| **所有环境** | JWT Bearer Token | `Authorization: Bearer <accessToken>`，密钥与 Nacos `gateway-service.yaml` 中 `jwt.secret` 一致 |

#### JWT 认证

通过聚宝赞登录接口获取 Token（示例）：

```
POST /buyer/api/v1/buyer/login
Content-Type: application/json

{"token": "test-token", "source": "JBZ"}
```

调用我方接口时携带：

```
POST /api/v1/chat/shop_001/stream
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
X-Tenant-Id: shop_001
X-Buyer-Id: buyer_123
Content-Type: application/json
```

我方校验逻辑：
1. 提取 `Authorization: Bearer <token>` 中的 JWT
2. 使用 HS256 + `jwt.secret` 验签
3. Gateway 注入的身份 Header（`X-Tenant-Id`、`X-Buyer-Id` 等）自动提取

认证失败时返回：

```json
{
  "detail": {"code": "GATEWAY_AUTH_REQUIRED", "message": "请求未经过 Gateway 认证，请通过内网 Gateway 访问"}
}
```

HTTP 状态码：`401`

### 2.3 请求追踪

建议在请求头中携带 `X-Request-ID`，用于问题排查时的日志关联：

```
X-Request-ID: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

如果不传，我方会自动生成并在响应头中返回。

### 2.4 通用响应结构

**成功响应**（各接口具体结构见下文）：

```json
{
  "success": true,
  "message": "操作描述"
}
```

**失败响应**：

```json
{
  "detail": "错误描述信息"
}
```

### 2.5 频率限制

| 接口类型 | 限制 |
|----------|------|
| 聊天/同步接口 | 同一 IP 每分钟 60 次 |
| 其他 API 接口 | 同一 IP 每分钟 120 次 |

> 💡 健康检查（`/api/v1/system/health`）和监控指标（`/api/v1/system/metrics`）接口不受限流影响，可放心高频调用。

超限时返回 HTTP `429 Too Many Requests`。

## 3. 消费者聊天接口

### 3.1 发起/继续对话（SSE 流式）

**接口地址**：`POST /api/v1/chat/{tenant_id}/stream`

**功能说明**：消费者发送消息，我方识别意图并处理后，通过 SSE（Server-Sent Events）流式返回 AI 回复。`session_id` 由聚宝赞端生成并管理，必须传入以保持上下文。

#### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `tenant_id` | string | ✅ | 商家租户 ID，如 `shop_001` |

#### 请求头

| Header | 必填 | 说明 |
|--------|------|------|
| `X-Gateway-Verified` | ✅ | Gateway 认证标识，值为 `true` |
| `Content-Type` | ✅ | `application/json` |
| `X-Request-ID` | ❌ | 请求追踪 ID |

#### 请求体

```json
{
  "message": "我的订单什么时候发货？",
  "session_id": "conv_abc123",
  "user_id": "user_456",
  "user_name": "张三",
  "channel": "app"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `message` | string | ✅ | 消费者消息内容，最长 4000 字符 |
| `session_id` | string | ✅ | 会话 ID，由聚宝赞端生成并管理，必须传入以保持上下文 |
| `user_id` | string | ✅ | 消费者用户 ID，用于关联订单/优惠券等查询 |
| `user_name` | string | ❌ | 消费者姓名 |
| `channel` | string | ❌ | 来源渠道：`app` / `miniapp` / `pc` / `h5`，默认 `unknown` |

#### 响应

**Content-Type**：`text/event-stream`

响应为 SSE 事件流，每个事件格式为 `data: {JSON}\n\n`。

**事件类型**：

| type | 说明 | 数据字段 | 是否必须处理 |
|------|------|----------|-------------|
| `status` | 节点处理状态 | `node`（节点名）、`action`（`start`/`end`） | ❌ 可选展示"正在思考..." |
| `text` | **AI 回复内容** | `content`（回复文本） | ✅ **核心数据，必须展示** |
| `done` | 对话完成 | `session_id`（会话 ID） | ✅ **必须保存 session_id** |
| `error` | 出错信息 | `message`（错误描述） | ✅ 展示错误提示 |

**完整 SSE 响应示例**：

```
data: {"type": "status", "node": "classify_intent", "action": "start"}

data: {"type": "status", "node": "classify_intent", "action": "end"}

data: {"type": "status", "node": "order_query_node", "action": "start"}

data: {"type": "text", "content": "为您查询到2条相关订单，以下是最近2条：\n\n[1] 订单号: ORD20240101890\n    商品: 燕麦片500g\n    金额: ¥38.8\n    状态: 已发货\n    物流单号: SF1234567890\n    快递公司: 顺丰速运\n    最新物流: 已到达北京分拨中心\n\n[2] 订单号: ORD20240102901\n    商品: 有机牛奶1L×6\n    金额: ¥68.0\n    状态: 待发货"}

data: {"type": "status", "node": "order_query_node", "action": "end"}

data: {"type": "done", "session_id": "conv_abc123"}
```

**错误响应示例**：

```
data: {"type": "error", "message": "请求处理超时，请稍后重试"}
```

#### 聚宝赞端 SSE 客户端实现参考

**JavaScript（浏览器）**：

```javascript
async function sendMessage(tenantId, message, sessionId, userId) {
  const response = await fetch(`/api/v1/chat/${tenantId}/stream`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Gateway-Verified': 'true',
    },
    body: JSON.stringify({
      message: message,
      session_id: sessionId,
      user_id: userId,
    }),
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n\n');
    buffer = lines.pop(); // 保留未完成的部分

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const data = JSON.parse(line.slice(6));

        switch (data.type) {
          case 'text':
            // 展示 AI 回复
            appendToChat(data.content);
            break;
          case 'done':
            // 保存 session_id，后续对话使用
            saveSessionId(data.session_id);
            break;
          case 'error':
            showError(data.message);
            break;
          case 'status':
            // 可选：展示"正在思考..."等状态
            showStatus(data.node, data.action);
            break;
        }
      }
    }
  }
}
```

```java
// 逐行读取 "data: {...}" 格式的 SSE 事件
// 解析 JSON 后转发给前端
```

#### 查询参数

| 参数 | 类型 | 必填 | 说明 |

#### 请求头

| Header | 必填 | 说明 |
|--------|------|------|
| `X-Gateway-Verified` | ✅ | Gateway 认证标识，值为 `true` |

#### 响应

```json
{
  "session_id": "conv_abc123",
  "messages": [
    {
      "role": "user",

      "role": "assistant",
```

## 4. 知识库同步接口

### 4.1 全量/增量同步

**接口地址**：`POST /api/v1/knowledge/sync/{tenant_id}/{kb_type}`

**功能说明**：聚宝赞知识变更时调用此接口推送数据。我方负责接收、切片、向量化、入库。支持两种模式：
- **全量同步**（`full`）：先清空该租户该类型的知识库，再写入新数据。适用于首次同步或大规模更新。
- **增量同步**（`incremental`）：追加/更新数据，不删除已有数据。适用于日常小范围更新。

> ⚡ 同步为实时处理：聚宝赞控制批量大小（约 10 条/批），调用后实时返回处理结果。

#### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `tenant_id` | string | ✅ | 商家租户 ID。平台公共知识库使用固定值 `public` |
| `kb_type` | string | ✅ | 知识库类型，见下表 |

**知识库类型（kb_type）**：

| kb_type | 说明 | 示例场景 |
|---------|------|----------|
| `faq` | 商家 FAQ 问答对 | 退货政策、配送范围等常见问题 |
| `product` | 商品知识 | 商品价格、规格、使用方法等 |
| `rule` | 规则文档 | 售后规则、会员规则等 |
| `public` | 平台公共知识库 | 平台保障政策、通用规则等（跨租户共享） |

#### 请求头

| Header | 必填 | 说明 |
|--------|------|------|
| `X-Gateway-Verified` | ✅ | Gateway 认证标识，值为 `true` |
| `Content-Type` | ✅ | `application/json` |

#### 请求体

```json
{
  "sync_type": "full",
  "items": [
    {
      "id": "faq_001",
      "content": "Q: 退货政策是什么？\nA: 收货后7天内可申请退货退款，请确保商品未使用且包装完好。",
      "metadata": {
        "category": "售后",
        "keywords": ["退货", "退款"],
        "status": "active"
      }
    },
    {
      "id": "faq_002",
      "content": "Q: 配送范围？\nA: 全国包邮，偏远地区（新疆、西藏）需补运费差价。",
      "metadata": {
        "category": "物流",
        "keywords": ["配送", "包邮"],
        "status": "active"
      }
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `sync_type` | string | ✅ | 同步类型：`full`（全量覆盖）/ `incremental`（增量追加） |
| `items` | array | ✅ | 知识条目列表 |
| `items[].id` | string | ✅ | 文档唯一 ID（聚宝赞系统中的 ID，用于后续增量更新和删除） |
| `items[].content` | string | ✅ | 文档内容（用于向量化检索，格式建议见下表） |
| `items[].metadata` | object | ❌ | 元数据，可包含分类、关键词、状态等任意键值对 |

**各 kb_type 的 content 格式建议**（遵循此格式可获得最佳检索效果）：

| kb_type | content 推荐格式 | 示例 |
|---------|-----------------|------|
| `faq` | `Q: {问题}\nA: {答案}` | `Q: 退货政策是什么？\nA: 收货后7天内可申请退货退款...` |
| `product` | `{商品名}(SKU:{sku}) {关键信息}` | `燕麦片500g(SKU:SKU001) 价格38.8元，规格500g/袋，保质期12个月` |
| `rule` | `{规则名称}\n{规则内容}` | `退换货规则\n1. 收货后7天内可申请退货\n2. 退货商品需未使用...` |
| `public` | `Q: {问题}\nA: {答案}` | `Q: 平台正品保障？\nA: 所有商品均为正品，支持假一赔十...` |

#### 响应

```json
{
  "success": true,
  "synced_count": 10,
  "kb_type": "faq",
  "tenant_id": "shop_001",
  "deleted_count": 0,
  "message": "同步完成，共处理 10 条",
  "timestamp": "2026-06-05T10:30:00Z"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | boolean | 是否处理成功 |
| `synced_count` | integer | 同步/新增的条数 |
| `kb_type` | string | 知识库类型 |
| `tenant_id` | string | 租户 ID |
| `deleted_count` | integer | 删除的条数 |
| `message` | string | 描述信息 |

### 4.2 批量增删

**接口地址**：`POST /api/v1/knowledge/sync/{tenant_id}/{kb_type}/batch`

**功能说明**：增量批量操作，支持同时新增/更新和删除文档。适用于日常小范围知识变更。

> ⚡ 实时处理，调用后立即返回结果。

#### 路径参数

同 [4.1 全量/增量同步](#41-全量增量同步)

#### 请求头

同 [4.1 全量/增量同步](#41-全量增量同步)

#### 请求体

```json
{
  "add": [
    {
      "id": "faq_new_001",
      "content": "Q: 新问题？\nA: 新答案内容",
      "metadata": {
        "category": "分类"
      }
    }
  ],
  "delete_ids": [
    "faq_old_001",
    "faq_old_002"
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `add` | array | ❌ | 新增/更新的知识条目列表（格式同 items） |
| `delete_ids` | array | ❌ | 要删除的文档 ID 列表 |

> `add` 和 `delete_ids` 至少传一个，不能同时为空。

#### 响应

```json
{
  "success": true,
  "kb_type": "faq",
  "tenant_id": "shop_001",
  "synced_count": 2,
  "deleted_count": 1,
  "message": "批量操作完成，新增 2 条，删除 1 条",
  "timestamp": "2026-06-05T10:30:00Z"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|

### 5.4 业务 API 调用模式

所有业务 API 调用统一通过 Nacos 服务发现进行，流程如下：

1. 我方服务（`kefu-service`）启动时自动注册到 Nacos 注册中心
2. 聚宝赞端业务服务（`merchant-service`）同样注册到同一 Nacos 注册中心
3. 我方调用聚宝赞业务 API 时，通过 Nacos 动态发现 `merchant-service` 实例，支持客户端负载均衡

**我方在 Nacos 中的注册信息**：

| 配置项 | 值 |
|--------|-----|
| 服务名 | `kefu-service` |
| 分组 | `DEFAULT_GROUP`（可配置） |
| 命名空间 | 空（public） |

**聚宝赞端需在 Nacos 中注册的服务名**：

| 业务 | Nacos 服务名 | 说明 |
|------|-------------|------|
| 全部业务 API | `merchant-service` | 订单、物流、商品、优惠券、用户画像均在此服务下 |

> 💡 服务名可通过 `.env` 中的 `MERCHANT_SERVICE_NAME` 变量自定义，Nacos 不可用时无法调用业务 API，需确保 Nacos 高可用。

### 5.5 业务 API 通用约定

1. **认证方式**：内网 VPN + Gateway 网关认证，由 Nacos 服务网格统一管理服务间认证
2. **响应结构**：统一为 `{code, message, data, timestamp, traceId, success}` 格式
3. **请求方法**：全部 POST（含查询类接口）
4. **超时时间**：我方默认设置 10 秒超时，超时后返回友好提示而非报错
5. **错误处理**：业务 API 不可用时，我方会降级为 LLM 生成回答或提示用户稍后重试
6. **数据脱敏**：手机号等敏感信息建议聚宝赞端已做脱敏处理（如 `138****5678`）

## 6. 错误码说明

### 6.1 HTTP 状态码

| 状态码 | 含义 | 触发场景 |
|--------|------|----------|
| `200` | 成功 | 请求处理成功 |
| `400` | 请求错误 | 参数格式错误、知识库类型无效等 |
| `401` | 认证失败 | Gateway 认证失败（缺少头 / 头值错误 / IP 不在白名单） |
| `403` | 禁止访问 | Admin API Key 无效 / 会话跨租户访问 / 会话跨用户访问 |
| `404` | 资源不存在 | 租户不存在、会话不存在 |
| `422` | 业务校验失败 | 消息为空、消息过长、检测到注入攻击、包含敏感词 |
| `429` | 请求过多 | 超出频率限制 |
| `500` | 服务器内部错误 | 我方服务异常 |
| `503` | 服务不可用 | LLM 服务不可用 |

### 6.2 业务错误码（422 响应的 detail 字段）

| 错误信息 | 触发场景 | 聚宝赞端处理建议 |
|----------|----------|-----------------|
| `消息不能为空` | message 为空或纯空格 | 提示用户输入内容 |
| `消息过长` | message 超过 4000 字符 | 前端限制输入长度 |
| `检测到不安全的输入` | 消息包含 Prompt 注入攻击 | 提示"请正常描述您的问题" |
| `输入包含违规内容` | 消息包含赌博/色情等敏感词 | 提示"请文明发言" |
| `无效的知识库类型: xxx` | kb_type 不在 faq/product/rule/public 中 | 检查 kb_type 参数 |
| `租户不存在: xxx` | tenant_id 未注册 | 检查 tenant_id 或先创建租户 |

### 6.3 SSE 流式错误

SSE 流中的 `error` 事件：

| message | 触发场景 | 聚宝赞端处理建议 |
|---------|----------|-----------------|
| `请求处理超时，请稍后重试` | AI 处理超过 120 秒 | 提示用户稍后重试 |
| `服务暂时不可用，请稍后重试` | LLM 服务异常 | 提示用户稍后重试 |

| 测试用户信息 | 用于联调的测试用户 ID、手机号等 |

### 7.4 联调前准备

1. **确认网络连通**：确保双方服务器网络互通，且双方均能访问 Nacos 注册中心
2. **Nacos 配置确认**：确认 Nacos 地址、命名空间、分组一致，双方服务均已注册
3. **创建租户**：聚宝赞提供 tenant_id 列表，我方在系统中注册
4. **同步初始知识**：聚宝赞调用知识同步接口推送首批知识数据
5. **验证业务 API**：双方确认业务 API 可正常通过 Nacos 服务发现调用

## 10. 注意事项与异常处理

### 10.1 关键注意事项

| 序号 | 事项 | 说明 |
|------|------|------|
| 1 | **session_id 必须传入** | `session_id` 由聚宝赞端生成并管理，必须传入，否则无法保持上下文 |
| 2 | **user_id 必须传入** | 不传 `user_id` 会导致订单查询、优惠券查询等功能无法关联用户 |
| 3 | **SSE 客户端必须支持流式读取** | 不能等全部响应完成后一次性处理，必须逐事件处理 |
| 4 | **知识同步为实时处理，聚宝赞控制批量大小（约 10 条/批），调用后立即返回结果 |
| 5 | **全量同步会先清空** | `sync_type=full` 会先清空该租户该类型的全部知识，再写入新数据，请确保 items 包含完整数据 |
| 6 | **知识库 ID 唯一性** | `items[].id` 在同一租户同一类型内必须唯一，重复 ID 会覆盖旧数据 |

### 10.2 异常处理机制

#### 我方异常处理

| 异常场景 | 我方处理方式 | 聚宝赞端感知 |
|----------|-------------|-------------|
| LLM 调用失败 | 自动重试 2 次，仍失败返回兜底话术 | SSE 返回"抱歉，服务暂时不可用" |
| 业务 API 超时 | 10 秒超时，返回友好提示 | SSE 返回"查询超时，请稍后重试" |
| 业务 API 不可用 | 降级为 LLM 生成回答，提示用户稍后重试 | 回复可能不够精确，但仍可回答知识库中已有的信息 |
| 意图识别失败 | 连续 2 次失败自动转人工 | SSE 返回转接话术 |
| SSE 流超时 | 120 秒总超时保护 | SSE 返回 error 事件 |
| 知识同步失败 | 返回错误信息，聚宝赞端根据返回结果判断是否重试 |

#### 聚宝赞端建议的异常处理

| 异常场景 | 建议处理方式 |
|----------|-------------|
| HTTP 401 | 检查 Gateway 认证头是否正确 |
| HTTP 422 | 根据 `detail` 字段提示用户修正输入 |
| HTTP 429 | 等待 1 分钟后重试 |
| HTTP 500/503 | 稍后重试，连续失败则提示用户联系人工客服 |
| SSE 连接中断 | 使用相同 `session_id` 和 `user_id` 重新发送消息 |
| SSE error 事件 | 展示错误提示，允许用户重试 |
| 业务 API 不可用 | 我方自动降级处理，聚宝赞端无需额外处理 |

### 10.3 数据安全

| 措施 | 说明 |
|------|------|
| Gateway 认证 | 所有接口均需经 Gateway 认证，防止未授权访问 |
| Prompt 注入检测 | 18 种注入模式 + Unicode 归一化 + 零宽字符过滤 |
| 敏感词审核 | 赌博/色情/毒品/诈骗等敏感词拦截 |
| 输出过滤 | 防止系统提示词泄露 |
| 日志脱敏 | 手机号、订单号自动掩码 |

### 10.4 常见问题

**Q1：session_id 如何生成和管理？**
A：`session_id` 由聚宝赞端生成并管理，每次新对话时由聚宝赞端生成唯一 ID，后续对话必须传入相同的 `session_id` 以保持上下文。

**Q2：session_id 过期了怎么办？**
A：我方目前不主动过期会话。如果传入的 `session_id` 对应的会话已不存在，我方会自动创建新会话。

**Q3：知识同步推送了重复的 id 会怎样？**  
A：增量模式下，相同 `id` 的文档会被覆盖更新；全量模式下会先清空再写入，不存在重复问题。

**Q4：消费者发送了敏感词会怎样？**  
A：我方会拦截并返回 HTTP 422，`detail` 包含"输入包含违规内容"。聚宝赞端可提示用户"请文明发言"。

**Q5：业务 API 暂时无法提供怎么办？**  
A：我方已做降级处理——业务 API 不可用时，会使用 LLM 基于知识库生成回答或返回提示信息。后续配置好 Nacos 服务注册后即可自动切换为真实数据。

**Q6：SSE 流式返回中途断了怎么办？**
A：使用相同的 `session_id` 和 `user_id` 重新发送消息即可继续对话。我方会保留对话上下文。
