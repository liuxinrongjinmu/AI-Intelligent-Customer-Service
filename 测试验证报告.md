## 一、测试判定标准

### 1.1 核心原则：断言驱动，非"能跑就行"

**判断测试是否成功的唯一标准是 `pytest` 的退出码和断言结果，而不是"代码能运行不出错"。** 具体而言：

| 判定项 | 标准 | 说明 |
|--------|------|------|
| 退出码 | 必须为 `0` | `pytest` 返回码 0 表示全部通过 |
| 失败数 | 必须为 `0` | `FAILED` 计数为 0 |
| 错误数 | 必须为 `0` | `ERROR` 计数为 0（收集阶段的异常也算） |
| 断言 | 每个测试至少 1 个具体断言 | 不允许空测试函数、不允许 `assert True` |
| 覆盖 | 每个函数的正向+反向路径都覆盖 | 正常输入 + 边界值 + 异常输入 |

### 1.2 禁止的"伪测试"模式

本项目严格禁止以下模式，经全量扫描确认**不存在**：

- `assert True` / `assert False`（无意义断言）
- 测试函数体为空（`def test_xxx(): pass`）
- 只调用函数不做断言
- 测试依赖外部服务且无 mock（导致 CI 不稳定）

### 1.3 每个测试用例必须满足的条件

一个合格的测试用例至少包含以下要素之一：

1. **正向验证**：给定合法输入，断言返回正确的具体值
2. **反向验证**：给定非法输入，断言抛出正确的异常类型和错误信息/错误码
3. **边界验证**：测试临界值（空字符串、None、最大长度、最小长度、网段边界等）
4. **回归验证**：针对已知 Bug 编写的用例，确保 Bug 不复发

## 四、测试质量评估

### 4.1 优势

- **零"伪测试"**：经过全量扫描，不存在 `assert True`、空函数体、无断言调用等无效测试
- **错误码精确匹配**：异常测试不仅验证异常类型，还验证具体的 HTTP 状态码和业务错误码（如 `GATEWAY_AUTH_REQUIRED`、`GATEWAY_IP_FORBIDDEN`）
- **Unicode 安全覆盖**：覆盖零宽字符、全角字符、BOM 等常见绕过手段
- **边界值完整**：每个校验函数都有空值、None、最大长度、刚好最大长度的测试
- **缓存行为验证**：LRU 淘汰、TTL 过期、访问移位、租户隔离等关键行为都有验证
