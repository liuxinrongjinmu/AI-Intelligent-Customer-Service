# 聚宝赞AI智能客服Agent系统 —— 项目需求说明文档

> 版本：v1.1 | 日期：2026-06-04 | 状态：MVP已实施（持续迭代中）

---

## 一、项目背景与目标

### 1.1 项目定位

打造一个**具备自主规划与执行能力的业务级AI客服Agent**，对外向商家提供智能客服SaaS服务。不同于传统的FAQ问答机器人，本系统的核心目标是实现"**理解 → 检索 → 执行 → 反馈**"的智能闭环交互。

### 1.2 与原型系统的关系

本系统的设计参考了 `聚宝赞AI客服系统原型终稿` 中的完整业务场景。原型稿覆盖了6个角色端、60+页面的完整业务闭环，MVP阶段聚焦其中最核心的一条链路：**商家管理知识库 → 消费者发起对话 → AI检索知识并回答**。

### 1.3 核心差异化能力

| 对比维度 | 传统FAQ客服 | 本Agent系统 |
|---------|-----------|-----------|
| 交互模式 | 关键词匹配 → 返回固定答案 | LLM理解语义 → RAG检索 → 流式生成 |
| 知识管理 | 固定问答对，人工维护 | 多源知识（FAQ + 文档），自动向量化 |
| 上下文 | 单轮无状态 | 多轮对话记忆，跨请求持久化 |
| 租户隔离 | 共享知识库 | 商家级全链路隔离（知识/配置/数据） |
| 扩展性 | 硬编码逻辑 | LangGraph编排，10节点可插拔 |
| 输出方式 | 一次性返回全部文本 | SSE流式逐字输出 |
| 业务集成 | 无 | Nacos服务发现 + 回调聚宝赞业务API |
| 安全防护 | 基本过滤 | Prompt注入检测 + 敏感词过滤 + 输出过滤 |

## 三、功能需求

### 3.1 MVP功能清单（已实施）

#### 3.1.1 多租户管理

- 平台初始化租户，生成唯一 `tenant_id` 和 `api_key`（`api_key` 仅用于管理接口认证，服务间接口走 Gateway 网关认证）
- 商家管理接口通过 `api_key` 访问，服务间接口通过 Gateway 网关认证
- 消费者通过 URL 路径中的 `tenant_id` 访问对应商家的AI客服
- 知识库按租户完全隔离（ChromaDB + SQLite 双重过滤）

#### 3.1.2 知识库管理（商家端 + 聚宝赞端同步）

| 功能 | 说明 |
|------|------|
| **FAQ 管理** | 商家可增删改查标准问答对（Q + A），支持分类和标签 |
| **文档上传** | 支持 PDF、TXT、Markdown 格式上传，自动切片向量化 |
| **知识查看** | 展示已录入的 FAQ 和文档列表，支持启用/禁用 |
| **知识检索测试** | 商家可在管理端测试搜索效果，输入问题查看召回结果 |
| **知识同步 API** | 聚宝赞端通过 API 推送知识变更，支持全量/增量同步、批量增删、清空知识库 |

#### 3.1.3 AI对话（消费者端）

| 功能 | 说明 |
|------|------|
| **聊天界面** | 清晰的消息气泡界面（用户消息靠右、AI消息靠左） |
| **流式输出** | AI回复逐字呈现（SSE），展示"正在输入"状态动画 |
| **意图识别** | 自动识别10种意图大类、42子类（含指代消解、实体抽取、知识库类型推荐） |
| **知识检索** | 混合检索（向量语义匹配 + 关键词文本匹配 + RRF 融合） |
| **多轮记忆** | 同一会话内保持上下文，支持 Token 预算自动裁剪 |
| **会话管理** | 聚宝赞端传入 session_id，已存在则继续对话，不存在则自动创建新会话 |

#### 3.1.4 业务 API 回调（已实施）

| 功能 | 说明 |
|------|------|
| **订单查询** | 通过 Nacos 发现 order-service，回调聚宝赞订单API |
| **物流查询** | 与订单查询联动（logistics_query 路由到同一节点） |
| **商品查询** | 优先调用聚宝赞商品API，通过 Nacos 服务发现调用 |
| **优惠券查询** | 通过 Nacos 发现 coupon-service |
| **用户画像查询** | 会员等级/积分/地址查询 |
| **转人工** | 用户主动转人工 / AI连续失败自动转人工 / 投诉自动转人工 |

#### 3.1.5 安全防护（已实施）

| 功能 | 说明 |
|------|------|
| **Prompt 注入检测** | 18种注入模式 + Unicode归一化 + 零宽字符过滤 |
| **敏感词过滤** | 赌博/色情/毒品/诈骗等敏感词拦截 |
| **输出过滤** | 防止系统提示词泄露 |
| **日志脱敏** | 手机号/订单号自动掩码 |
| **管理接口 API Key 认证** | `hmac.compare_digest` 时序安全比较（仅管理接口） |
| **限流控制** | 聊天/同步 60次/分，API 120次/分，分桶独立 |

#### 3.1.6 意图分类体系

| 优先级 | 意图大类 | 子类数 | 说明 |
|--------|---------|-------|------|
| 1 | `human_service` | 5 | 转人工（用户要求/投诉/情绪/敏感操作/AI失败） |
| 3 | `order_query` | 5 | 订单查询 |
| 3 | `logistics_query` | 4 | 物流查询 |
| 3 | `product_query` | 5 | 商品咨询 |
| 3 | `coupon_query` | 4 | 优惠券咨询 |
| 3 | `account_query` | 4 | 账户查询 |
| 4 | `knowledge_query` | 7 | 通用知识检索 |
| 5 | `greeting` | 4 | 闲聊问候 |
| 5 | `feedback` | 2 | 反馈确认 |
| 5 | `other` | 2 | 兜底 |

## 四、非功能需求

### 4.1 性能要求

| 指标 | MVP目标 | 当前状态 |
|------|--------|---------|
| 首次响应时间 | < 3秒 | ✅ 已达标 |
| 流式首token时间 | < 2秒 | ✅ 已达标 |
| 知识检索延迟 | < 500ms | ✅ 已达标 |
| 并发会话数 | ≥ 50 | ✅ 已达标 |

### 4.2 安全要求

| 要求 | 说明 | 状态 |
|------|------|------|
| 租户数据隔离 | 所有查询必须携带 `tenant_id` 过滤 | ✅ |
| API鉴权 | Gateway 网关认证（X-Gateway-Verified + IP 白名单） | ✅ |
| 敏感信息保护 | Gateway 网关认证 + 管理接口 API Key，日志脱敏 | ✅ |
| LLM输出约束 | Prompt 中约束不编造事实，不泄露其他租户信息 | ✅ |
| Prompt注入防护 | 18种注入模式检测 + Unicode归一化 | ✅ |
| 敏感词过滤 | 赌博/色情/毒品/诈骗拦截 | ✅ |

### 4.3 可维护性要求

| 要求 | 说明 | 状态 |
|------|------|------|
| 模块化 | Agent 节点独立定义，新增意图只需加节点 + 条件边 | ✅ |
| 可测试 | 单元测试 + 全链路集成测试 | ✅ |
| 可观测 | 关键环节日志记录 + 请求链路追踪（X-Request-ID） | ✅ |
| 服务治理 | Nacos 服务注册/心跳/发现/负载均衡 | ✅ |

## 五、约束条件

### 5.1 技术约束

| 约束 | 说明 |
|------|------|
| 开发环境 | Windows 本地（团队统一Windows） |
| 编程语言 | Python 3.11+ |
| LLM | deepseek-chat（统一模型，不按租户区分） |
| Embedding | BAAI/bge-small-zh-v1.5（本地运行） |
| 向量数据库 | ChromaDB（MVP过渡，目标迁移到Milvus） |
| 业务数据库 | SQLite + aiosqlite（WAL模式，后续迁移到PostgreSQL） |
| 服务注册发现 | Nacos（与聚宝赞端统一服务治理，所有业务API调用均通过Nacos服务发现） |
| 前端 | FastAPI + Jinja2 服务端渲染（复用原型稿风格） |
| 部署形态 | Windows 本地开发 / Docker 容器化部署 |

### 5.2 业务约束

| 约束 | 说明 |
|------|------|
| 对话模式 | MVP仅支持1对1对话（用户 ↔ AI） |
| 知识格式 | 支持 FAQ 问答对 和 文档文件（PDF/TXT/MD） |
| 检索方式 | 向量语义检索 + 关键词文本匹配 + RRF 融合（向量权重 0.7，关键词权重 0.3） |
| 业务API | 通过 Nacos 服务发现调用聚宝赞端业务接口 |

## 六、验收标准

### 6.1 MVP验收用例

| # | 用例 | 预期结果 | 状态 |
|---|------|---------|------|
| 1 | 商家上传FAQ：Q="退货政策是什么" A="7天无理由退货" | 知识存入ChromaDB，可检索到 | ✅ |

| 4 | 消费者连续问"那运费谁出？"（上文是退货话题） | AI结合上文理解指代，在知识库中检索并回答 | ✅ |

| 9 | 聚宝赞端通过API同步知识 | 知识成功切片、向量化、入库 | ✅ |
| 10 | 消费者查询订单 | 通过Nacos发现order-service，回调获取真实订单数据 | ✅ |
| 11 | 消费者发送注入攻击 | Prompt注入检测拦截，返回422 | ✅ |
| 12 | 商品API不可用时查询商品 | 自动降级为知识库检索 | ✅ |
| 13 | AI连续2次失败 | 自动转人工 | ✅ |
